## The trick that makes LLMs talk faster — without dumbing them down

A small model guesses what a large model *would* say, and the large model checks the guesses in bulk. When the guesses are right (and they usually are), you skip the slow sequential generation entirely.

**What it is:** Large language models generate text one token at a time — each token requires a full forward pass before the next can begin. Speculative decoding breaks this bottleneck by introducing a **draft model** (small, fast) that proposes the next *K* tokens speculatively, and a **target model** (large, accurate) that verifies all *K* tokens in a single batched forward pass. Accepted tokens are kept; rejected ones are discarded and regenerated by the target model. The key mathematical guarantee: the output distribution is *identical* to running the target model alone, so there is **zero quality degradation**.

**Why it matters:** LLM inference is memory-bandwidth-bound — GPUs sit idle between token generations because they're waiting on memory, not compute. Speculative decoding exploits this idle compute by verifying multiple tokens at once, achieving 2-5× throughput improvements on real hardware. This directly translates to lower latency in chatbots, faster code-completion, and reduced serving costs at scale. Every major inference framework (vLLM, TensorRT-LLM, SGLang) now supports it out of the box.

**Example or insight:** The acceptance-rejection mechanism borrows from rejection sampling in statistics. Given a draft model probability $q(x)$ and target model probability $p(x)$ for a proposed token $x$:

$$
P(\text{accept}) = \min\left(1, \frac{p(x)}{q(x)}\right)
$$

If the draft token is rejected, a corrected token is sampled from the residual distribution:

$$
p'(x) = \frac{\max(0, \; p(x) - q(x))}{\sum_{x'} \max(0, \; p(x') - q(x'))}
$$

This guarantees the final output follows $p(x)$ exactly. Here's the intuition in pseudocode:

```python
def speculative_decode(draft_model, target_model, prompt, K=5):
    while not done:
        # Draft: propose K tokens quickly
        draft_tokens = draft_model.generate(prompt, num_tokens=K)

        # Verify: score all K tokens in one target forward pass
        target_probs = target_model.forward(prompt + draft_tokens)

        # Accept/reject each token via rejection sampling
        for i, token in enumerate(draft_tokens):
            r = random.uniform(0, 1)
            if r < min(1, target_probs[i][token] / draft_probs[i][token]):
                prompt.append(token)  # accept
            else:
                # Sample correction from residual distribution
                prompt.append(sample_residual(target_probs[i], draft_probs[i]))
                break  # re-draft from here

    return prompt
```

The "aha!" moment: you get **K+1 tokens** from **one** expensive forward pass instead of one token per pass. The better aligned the draft model is to the target, the higher the acceptance rate — and the closer you get to that theoretical K× speedup.
